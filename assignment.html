Assignment: Student–Teacher Web Application

Design and implement a production-ready web app for a school with distinct student and teacher roles.

Assigned to: Abdul-Moiz
Next-js Assestment

Objective

Create a role-based web application for school use

Build a Student–Teacher system where teachers can create quizzes and manage attendance, and students can take quizzes and check their marks.

Roles & Features

Teacher

  • Signup/Login: Email, name, and class they teach.
  • Quiz Management: Create quizzes for a specific date & time (questions, types, options).
  • Evaluation: View student attempts and grade them.
  • Attendance: View student check-ins.

Student

  • Signup/Login: Email, name, and class.
  • Quizzes: Attempt quizzes assigned by teachers.
  • Attendance: Daily check-in for attendance.
  • Marks: View quiz marks and results.

Technical Requirements

  • Frontend: Next.js (Middleware & API Routes required), Redux, Tailwind CSS for styling.
  • Backend/Database: Firebase (authentication, Firestore/Realtime DB, storage as needed).
  • Restrictions: No AI tools. Only allowed extra libraries: moment() and react-calendar.

Deliverables

  • Working web application implementing required features.
  • Documentation / README with setup instructions, features, and tech-stack notes.
  • Optional: demo video or hosted link (if available).

Evaluation Criteria

Clarity, functionality, and code quality
  • Functionality (40%): All required features work (signup, quizzes, grading, attendance).
  • Code Quality & Architecture (25%): Clean, modular code using Next.js best practices and Redux for state management.
  • UI/UX & Styling (15%): Responsive, accessible, and well-styled using Tailwind.
  • Documentation (10%): Clear README with setup and usage instructions.
  • Extra (10%): Bonus for tests, CI, or additional useful features (notifications, export reports, etc.).
